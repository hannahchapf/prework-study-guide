HTML

  • HTML has declarations, head elements, and body elements
  • Attributes are defined within elements
  • Order matters and elements can be nested in each other meaning there is a parent child relationship
  • Head elements are non-visible metadata, such as style, base, link, or title
  • Body elements are visible, such as header, main, or footer, and elements within these sections such as img or section

CSS

  • Styling can be done via inline CSS, internal CSS style sheets, and external CSS style sheets, but the latter is most common
  • External CSS style sheets are kept separately but linked to in the head element
  • CSS rules on style sheets are defined as property-value pairs (declarations) and form declaration blocks together for whatever element they are formatting
  • * can be used as a wildcard to format all elements on the page

Git

How to push new code to GitHub:

  1. git status: checks current branch
  2. git checkout -b branch-name: creates new branch and moves to it
  3. git add -A: adds changed files to branch
  4. git commit -m "message to add": commit additions to branch with context message
  5. git pull origin main: pulls source to ensure latest changes are applied
  6. git push origin branch-name: push commit to branch

Javascript

  • Control flow is important, which is the order the code is executed in
  • The key concepts were:
  1. Variables: declare using var variable-name = "variable"
  2. Conditionals: write using if (first condition) {code} else if (second condition) {code} syntax
  3. For loops: define such as for (var x=0; condition less than array.length; x++) {code}
  4. Functions: declare as function functionName() {code} and call with function name
